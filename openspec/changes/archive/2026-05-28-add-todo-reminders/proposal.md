## Why

项目原规划已经完成到阶段 9，下一步是阶段 10：为登录用户提供私密 TodoList，并在进入网站时弹窗提醒到期或即将到期的待办。该功能补齐个人效率工具能力，让博客系统不仅能记录内容，也能提醒行动。

## What Changes

- 新增登录用户专属 TodoList 后端能力：
  - 创建、读取、更新、删除待办。
  - 标记待办完成或未完成。
  - 支持分类、截止时间、内容描述。
  - 提供到期/今日到期待办提醒查询接口。
- 新增 `/todo` 前端页面：
  - 列表展示当前用户自己的待办。
  - 支持新增、编辑、删除、完成状态切换。
  - 支持分类输入或选择、截止时间选择。
- 新增全局页面弹窗提醒：
  - 登录用户进入网站后自动查询提醒。
  - 对已到期、今日到期或即将到期的未完成待办弹窗提示。
- 保持权限边界：
  - Todo 强制仅自己可见。
  - 游客不可访问 Todo 页面或 Todo API。

### New Capabilities
- `todo-reminders`: 覆盖私密待办 CRUD、完成状态、截止时间、分类和页面弹窗提醒。

## Impact

- 后端：新增 Todo 数据表、schema、service、API router，并接入总路由。
- 前端：实现 `/todo` 页面，接入现有路由守卫和导航；新增全局提醒弹窗组件或布局逻辑。
- 数据库：SQLite 初始化新增 `todos` 表和索引。
- 验证：需要覆盖 API 权限隔离、CRUD、提醒查询、前端页面交互和弹窗行为。
