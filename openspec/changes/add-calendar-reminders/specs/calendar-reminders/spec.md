## ADDED Requirements

### Requirement: 私密日历事件隔离
系统 SHALL 将日历事件强制绑定到当前登录用户，任何用户只能访问自己的日历事件。

#### Scenario: 游客访问日历 API
- **WHEN** 未登录访客请求日历事件 API
- **THEN** 系统 MUST 拒绝请求并要求登录

#### Scenario: 用户查看日历事件
- **WHEN** 登录用户请求日历事件
- **THEN** 系统 MUST 只返回该用户自己的事件

#### Scenario: 用户修改他人事件
- **WHEN** 登录用户尝试更新或删除不属于自己的日历事件
- **THEN** 系统 MUST 不返回他人数据，并按不存在处理

### Requirement: 日历事件 CRUD
系统 SHALL 支持登录用户创建、读取、更新和删除自己的日历事件。

#### Scenario: 创建事件
- **WHEN** 登录用户提交标题、日期、事件类型、备注和年度重复设置
- **THEN** 系统 MUST 创建一条属于该用户的日历事件

#### Scenario: 更新事件
- **WHEN** 登录用户修改自己的日历事件
- **THEN** 系统 MUST 保存修改并更新该事件的更新时间

#### Scenario: 删除事件
- **WHEN** 登录用户删除自己的日历事件
- **THEN** 系统 MUST 移除该事件，后续列表不再返回

### Requirement: 月历视图事件展示
系统 SHALL 在 `/calendar` 页面提供月历视图，并标注当前月份内的事件。

#### Scenario: 打开日历页面
- **WHEN** 登录用户打开 `/calendar`
- **THEN** 页面 MUST 展示当前月份的月历网格

#### Scenario: 月份内存在事件
- **WHEN** 当前月份某日期存在用户自己的日历事件
- **THEN** 页面 MUST 在对应日期格展示事件标记或标题

#### Scenario: 切换月份
- **WHEN** 登录用户切换到上一个月或下一个月
- **THEN** 页面 MUST 展示目标月份，并加载该月份可显示的事件

### Requirement: 年度重复事件
系统 SHALL 支持每年重复的日历事件，并在月历和提醒查询中按目标年份显示。

#### Scenario: 年度重复事件跨年显示
- **WHEN** 用户创建一个每年重复的生日或纪念日事件
- **THEN** 系统 MUST 在后续年份对应月日继续显示该事件

#### Scenario: 非重复事件不跨年显示
- **WHEN** 用户创建一个非年度重复事件
- **THEN** 系统 MUST 只在原始事件日期所在年月显示该事件

#### Scenario: 闰日年度重复
- **WHEN** 用户创建 2 月 29 日的每年重复事件，且目标年份不是闰年
- **THEN** 系统 MUST 将该事件映射到目标年份 2 月 28 日显示和提醒

### Requirement: 未来 7 天事件提醒查询
系统 SHALL 提供提醒查询能力，返回当前用户未来 7 天内发生的日历事件。

#### Scenario: 查询近期事件提醒
- **WHEN** 登录用户请求日历提醒
- **THEN** 系统 MUST 返回该用户未来 7 天内发生的事件

#### Scenario: 年度重复事件进入提醒窗口
- **WHEN** 每年重复事件的本年度或下一年度发生日期落在未来 7 天内
- **THEN** 系统 MUST 在提醒结果中返回该事件

#### Scenario: 排除提醒窗口外事件
- **WHEN** 日历事件的显示日期不在未来 7 天内
- **THEN** 系统 MUST NOT 在提醒结果中返回该事件

### Requirement: 统一提醒中心
系统 SHALL 使用统一提醒弹窗展示 Todo 提醒和日历提醒，避免多个全局弹窗同时出现。

#### Scenario: 同时存在 Todo 和日历提醒
- **WHEN** 登录用户进入任意应用页面，且同时存在 Todo 提醒和日历事件提醒
- **THEN** 页面 MUST 展示一个统一提醒弹窗，并分组列出两类提醒

#### Scenario: 仅存在日历提醒
- **WHEN** 登录用户进入任意应用页面，且只存在日历事件提醒
- **THEN** 页面 MUST 展示统一提醒弹窗，并展示日历提醒分组

#### Scenario: 游客进入网站
- **WHEN** 未登录访客进入任意应用页面
- **THEN** 页面 MUST NOT 请求私密日历提醒，也 MUST NOT 展示日历提醒弹窗

#### Scenario: 关闭统一提醒弹窗
- **WHEN** 登录用户关闭统一提醒弹窗
- **THEN** 页面 MUST 关闭弹窗，并在本次页面生命周期内不反复弹出同一批提醒
