## Why

当前顶部搜索和点赞等按钮主要依赖文字或字符符号，视觉识别度不够稳定。用户希望关键按钮使用更清晰的矢量图形，例如搜索放大镜、爱心点赞、展开箭头等，以提升界面质感和可读性。

同时，上一轮搜索按“博客 + 快记”实现，是基于当时“搜索只搜索博客或者快记”的表述。但用户现在明确指出需要搜索首页的“快写”内容。快写对应后端 `quick_posts`，不是私有快记 `quick_notes`，因此搜索能力需要从“博客 + 快记”调整为“博客 + 快写”。

首页布局也需要重新整理：快写不应该压在博客上方，而应该回到右侧栏。整体空间应更充分利用：左侧主区域展示博客，右侧上方放日历，日历下方展示快写；博客位于左侧快写发送栏下方，右侧快写列表尽量填满右侧栏，减少左右空白。

## What Changes

- 按钮图标矢量化：
  - 搜索入口使用矢量放大镜图标。
  - 点赞使用矢量空心/实心爱心图标，不再依赖普通文本字符。
  - 日历展开按钮使用矢量箭头图标。
  - 图标按钮保留无障碍文本或 `aria-label`。

- 搜索范围修正：
  - 搜索范围从“博客 + 快记”改为“博客 + 快写”。
  - 博客搜索仍匹配标题、标签、分类、简介。
  - 快写搜索匹配快写正文内容。
  - 搜索结果分区显示“博客结果”和“快写结果”。
  - 搜索结果布局必须保持左右关系：博客相关内容显示在左侧，快写相关内容显示在右侧。
  - 搜索结果不应改成上下堆叠关系，因为上下翻阅会降低对比和浏览效率。
  - 不搜索快记、待办或日历。

- 首页布局重排：
  - 首页恢复为左右主布局。
  - 左侧主区域展示博客内容。
  - 左侧上方保留博客相关入口或发布/阅读引导，博客列表显示在其下方。
  - 右侧栏上方展示小月历。
  - 右侧栏下方展示快写列表。
  - 快写发送栏应与快写列表所在区域关系更清晰，避免把快写压到博客上方。
  - 页面内容宽度应更充分利用，减少左右侧明显空白。
  - 响应式压缩时可以减少左右边距、缩小栏宽、压缩组件内容，但不能改变主要模块的左右相对位置。
  - 待办入口必须始终保持在页面最左侧，不能在窄屏时变成压住其他功能的整块内容框。
  - 博客必须始终位于左侧主区域。
  - 快写必须始终位于右侧区域。
  - 如果宽度不足以完整显示日历月历，日历可以压缩成只显示“日历”两个字或等价的极简入口，不再强行显示月历。

## Capabilities

### New Capabilities

- `vector-icon-controls`: 统一关键按钮的矢量图标表现，包括搜索、点赞和日历展开。

### Modified Capabilities

- `global-content-search`: 搜索目标从博客 + 快记修正为博客 + 快写，结果分区对应调整，并保持博客左侧、快写右侧的结果布局。
- `home-dashboard-layout`: 首页布局改为左侧博客、右侧日历 + 快写，充分利用屏幕宽度。
- `home-dashboard-layout`: 响应式时保持待办、博客、快写的相对方位不变，只压缩边距、宽度和日历展示密度。
- `quick-post-detail`: 快写搜索结果可进入快写详情。

## Impact

- 后端：
  - 修改 `backend/app/schemas/search.py`，将 `quick_notes` 结果改为 `quick_posts` 结果。
  - 修改 `backend/app/api/search.py`，搜索快写而不是快记。
  - 修改 `backend/app/services/quick_post_service.py`，新增快写搜索逻辑并沿用现有可见性规则。
  - 可能移除或停止使用上一轮新增的 `quick_note_service.searchQuickNotes`。

- 前端：
  - 修改 `frontend/src/pages/SearchPage.jsx`，将“快记结果”改为“快写结果”，点击进入 `/quick-posts/:id`，并让博客结果位于左侧、快写结果位于右侧。
  - 修改 `frontend/src/layouts/AppLayout.jsx`，搜索按钮改用矢量图标。
  - 修改 `frontend/src/components/LikeButton.jsx`，使用矢量爱心图标。
  - 修改 `frontend/src/pages/HomePage.jsx`，重新排布左侧博客与右侧日历/快写。
  - 修改 `frontend/src/styles/global.css`，补充矢量图标按钮、搜索页和首页布局样式，并确保窄屏压缩时不改变博客/快写/待办的相对位置。

- OpenSpec：
  - 新增 `vector-icon-controls` 规格。
  - 修改 `global-content-search`、`home-dashboard-layout`、`quick-post-detail` 相关规格。

## Non-Goals

- 不搜索快记、待办或日历。
- 不删除快记功能本身。
- 不引入大型图标库；优先使用内联 SVG 或轻量本地组件。
- 不改变快写、博客、评论的点赞 API 行为。
- 不改变快写和博客的可见性权限规则。
