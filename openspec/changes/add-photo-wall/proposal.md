## Why

项目原规划的下一阶段是照片墙独立模块，用于集中展示用户上传过的图片资源。当前 `/photo` 已有公开展示页面雏形，但缺少登录用户自己的照片管理、公开/私密权限切换、预览复制和删除能力，需要补齐资源管理闭环。

## What Changes

- 为登录用户提供个人照片墙管理能力，可查看自己上传或归集的照片。
- 游客和其他用户只能浏览公开照片，私密照片仅照片拥有者可见。
- 单张照片支持大图预览、复制图片链接、公开/仅自己可见切换。
- 照片拥有者可以删除自己的照片，删除后列表不再返回该照片。
- 管理员预留全站照片管理接口能力，本阶段至少保证后端权限边界可扩展。
- 不在本变更中改造 Markdown 图片上传或头像上传流程；如现有图片归集数据不足，本阶段先支持手动/接口级照片记录管理，并为后续自动归集预留字段。

## Capabilities

### New Capabilities

- `photo-wall`: 登录用户照片管理、公开照片浏览、单张预览、链接复制、可见性切换、删除和权限隔离。

### Modified Capabilities

- 无。

## Impact

- 后端：新增或扩展照片数据表、照片 schema、照片 service、照片 API，并注册到统一 API router。
- 前端：替换 `/photo` 占位/静态页面为真实照片墙页面，增加登录态管理操作、游客公开浏览和预览弹窗。
- 样式：补充照片网格、瀑布流近似布局、预览弹窗、可见性控件和操作按钮样式。
- 验证：需要覆盖游客公开浏览、登录用户私密隔离、拥有者权限、删除、复制链接和管理员扩展边界。
