## Why

核心功能已经基本落齐，但各阶段增量开发后，页面密度、按钮层级、弹窗、表单、空状态和响应式细节存在风格不完全一致的风险。现在进入原规划阶段12，需要统一全站 UI 体验，并对关键流程和权限隔离做一次系统复测。

## What Changes

- 统一全站视觉细节：导航、面板、表单、按钮、卡片、弹窗、列表、空状态、错误提示和加载状态。
- 优化移动端和窄屏表现，确保主要页面内容不重叠、不溢出、按钮文字不挤压。
- 梳理全局 CSS，减少重复样式和不一致的交互反馈，但不引入新的设计系统依赖。
- 对已实现模块做一次关键流程复测：登录注册、个人中心、关注、文章/草稿、快写/快记、Todo、日历、照片墙。
- 对游客、普通登录用户、资源拥有者、非拥有者和管理员边界做权限复测。
- 修复复测过程中发现的明显 UI bug、权限 bug 和流程阻断问题。
- 不新增业务模块，不改变已确认的核心产品功能范围。

## Capabilities

### New Capabilities

- `global-ui-quality`: 全站 UI 一致性、响应式质量、状态呈现、关键流程稳定性和权限复测。

### Modified Capabilities

- 无。

## Impact

- 前端：主要影响 `frontend/src/styles/global.css`、布局组件和各页面的细节状态呈现；必要时小幅调整页面 JSX 结构以统一操作区、空状态和弹窗。
- 后端：原则上不做业务扩展；仅修复复测中发现的权限或流程缺陷。
- OpenSpec：新增阶段12质量收敛变更，后续设计和任务清单会列明每个允许修改的文件范围。
- 验证：需要运行 lint/build，并使用 API 与 in-app browser 覆盖核心用户流程和权限边界。
