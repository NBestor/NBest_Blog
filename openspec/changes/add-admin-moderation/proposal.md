## Why

当前项目只有 `admin` 角色字段和少量资源管理权限，管理员登录后缺少可见、可操作、可验证的管理入口。管理员模式需要成为真实可用的内容与用户治理能力，支持处理违规用户资料、文章、快写、评论等基础场景。

## What Changes

- 新增管理员专属入口：
  - 普通用户和游客完全看不到管理员管理前端 UI。
  - 管理员登录后，页面最上方导航栏新增“管理”入口。
  - 主管登录后同样显示“管理”入口，并在管理页内额外显示管理员任命与管理员管理能力。
  - 普通用户和游客即使直接访问管理员路由，也必须被拒绝或重定向。
- 新增主管/超级管理员账号：
  - 固定用户 ID 为 `0`。
  - 固定账号名为 `NBest`。
  - 初始密码暂定为 `NBest666`。
  - 主管账号拥有最高权限，用于管理所有管理员。
- 新增管理员概览页面，用于集中查看用户、文章、快写、评论和照片等需要治理的内容。
- 新增管理员任命与管理能力：
  - 主管可以将普通用户任命为管理员。
  - 主管可以撤销管理员身份。
  - 主管可以查看和管理所有管理员。
  - 普通管理员不能任命管理员，不能撤销管理员，不能管理其他管理员。
  - 任何管理员都不能删除、禁用或降级主管账号。
- 新增用户管理能力：
  - 管理员可以查看用户列表和用户基础资料。
  - 管理员可以删除普通用户。
  - 普通管理员不能删除自己，不能删除其他管理员，不能删除主管。
  - 主管可以管理管理员账号，但不能删除主管账号自身。
  - 管理员可以处理违规用户名/昵称，例如强制重命名为合规占位名称。
  - 管理员可以移除违规头像，使用户回到默认头像状态。
- 新增内容管理能力：
  - 管理员可以删除博客文章。
  - 管理员可以删除快写。
  - 管理员可以删除评论。
  - 管理员可以继续管理照片，包括删除违规照片。
- 新增后端管理员 API 权限边界：
  - 所有管理员 API 必须要求登录且角色为 `admin`。
  - 普通用户访问管理员 API 必须被拒绝。
  - 涉及管理员任命、撤销管理员、管理管理员账号的 API 必须要求主管身份。
  - 管理员删除或修改资源后，普通列表和详情接口不应再返回被删除内容。
- 新增前端管理员页面的状态呈现：
  - 加载、空状态、成功提示、失败提示和二次确认。
  - 删除类高风险操作必须有明确确认。
  - 管理入口、管理页面和管理操作按钮必须按用户身份显示；普通用户不应看到任何管理员 UI 暗示。
- 不新增新的业务依赖，不引入新的 UI 框架。
- 不改变普通用户现有写作、快写、照片墙、Todo、日历和关注流程。

## Capabilities

### New Capabilities

- `admin-moderation`: 管理员用户与内容治理能力，包括主管账号、管理员任命、管理员入口、用户管理、违规资料处理、文章/快写/评论/照片删除、权限边界和前端管理页面。

### Modified Capabilities

- `user-identity-navigation`: 管理员和主管登录后顶部导航需要显示“管理”入口，普通登录用户和游客不显示任何管理员入口或管理 UI。

## Impact

- 后端：
  - 可能新增 `backend/app/api/admin.py` 或等价管理员路由模块。
  - 可能修改 `backend/app/api/router.py` 注册管理员路由。
  - 可能扩展 `backend/app/services/user_service.py`、`article_service.py`、`quick_post_service.py`、`comment_service.py`、`photo_service.py` 中的管理员查询和删除能力。
  - 可能新增或复用管理员角色校验 helper，并增加主管身份校验。
  - 可能需要数据库初始化逻辑确保 ID 为 `0` 的 `NBest` 主管账号存在。
- 前端：
  - 可能新增 `frontend/src/pages/AdminPage.jsx` 或拆分的管理员页面组件。
  - 修改 `frontend/src/routes/route-config.js` 和 `frontend/src/layouts/AppLayout.jsx`，让管理员入口只对管理员可见。
  - 修改 `frontend/src/App.jsx` 增加管理员路由保护。
  - 修改 `frontend/src/styles/global.css` 补充管理员页面所需的表格/列表/危险操作样式。
- OpenSpec：
  - 新增 `openspec/changes/add-admin-moderation/specs/admin-moderation/spec.md`。
  - 修改 `user-identity-navigation` 的变更规格，增加管理员入口可见性要求。
